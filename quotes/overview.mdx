Quotes (called *offertes* throughout the app) let you send professional proposals to clients before any work begins. Once a client accepts, you can convert the quote to an invoice with one click—no re-entering data.

## Quote lifecycle

A quote moves through a fixed set of statuses from creation to payment:

```
Draft → Sent → Accepted → Invoice
                ↓
             Declined
                ↓
             Expired
```

| Status | What it means |
|---|---|
| **Draft** | The quote is being prepared. You can still edit all fields. |
| **Sent** | The quote has been emailed to the client. A personal acceptance link is active. |
| **Accepted** | The client has accepted online. Ready to be converted to an invoice. |
| **Declined** | The client declined. You can see their reason (if provided) and create a new quote. |
| **Expired** | The validity date has passed without a response. |
| **Archived** | Manually archived. The quote remains visible but is hidden from your default view. |

<Note>
A quote can only be accepted or declined while its status is **Sent** and the validity date has not passed. Once expired, the client's acceptance link becomes inactive.
</Note>

## Quote numbering

Quote numbers are generated automatically in the format `OFF[year]-[sequence]`, for example `OFF2025-00001`. You can customise the prefix in your administration settings, including placeholders like `[jaar]` and `[maand]`.

## Validity period

Every quote has a validity date (`geldig_tot`). By default, TRONVoice sets this based on the number of days configured in your administration settings. You can override it per quote when creating or editing.

<Tip>
Set a realistic validity period to create urgency without pressure. 14–30 days is common for most service-based proposals.
</Tip>
