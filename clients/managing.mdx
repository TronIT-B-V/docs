A client record is more than a name and address. Once you open a client, you get a dedicated workspace for everything connected to that relationship — invoices, files, notes, tasks, and settings that override your administration defaults.

## The client detail page

Open a client by clicking its name anywhere in the **Klanten** (Clients) list. The detail page brings together:

- **Company details** — name, KvK number, VAT number, address, and contact information.
- **Invoice statistics** — totals for invoiced, paid, and outstanding amounts.
- **Recent invoices** — the latest invoices and subscriptions for this client.
- **Tabs** for **Notities** (Notes), **Taken** (Tasks), and **Dossier** — covered below.

From the detail page you can also edit the client, send a SEPA mandate request, or create a new invoice with the client pre-selected.

## KvK company lookup

When you add a client, use the KvK search at the top of the form to find a registered Dutch company by name or registration number. Selecting a result automatically fills in the **company name**, **KvK number**, **address**, **postcode**, and **city**.

<Tip>
  The KvK lookup works best with official trade names. If a search returns no results, enter the details by hand — every field can still be edited.
</Tip>

## BTW number validation

When you enter a **BTW-nummer** (VAT number) for a client in an EU country, TRONVoice validates it in real time against the **EU VIES service**. If the number is not valid or not registered, the form blocks the save and shows the reason returned by VIES.

VAT treatment is also detected automatically from the client's country:

- **Same country as your administration** — standard VAT rates apply.
- **Another EU country** — VAT is set to *reversed* (BTW verlegd), as required for intra-community supplies.
- **Outside the EU** — VAT is set to *exempt* (BTW vrijgesteld), for export deliveries.

You can override this on the client form. If you change a client's VAT settings later, TRONVoice automatically syncs the new settings to that client's active subscriptions.

## The company logo

When you save a client with a **website** or **email address**, TRONVoice automatically fetches the company logo from that domain. The logo appears on the client record and can be used on invoices.

The logo refreshes automatically when the website or email address changes. Logos that were entered manually are left untouched.

<Note>
  Logo fetching never blocks saving a client. If the logo cannot be retrieved, the client is still saved and you can try again by updating the website later.
</Note>

## Notes

The **Notities** (Notes) tab holds internal CRM notes about the client — call summaries, agreements, or reminders. Notes are visible only to your team and never appear on invoices or emails.

- Click **New note** to add a note (minimum 3 characters).
- Each note records who wrote it and when.
- You can delete a note you created yourself.

The client form also has a separate free-text **Notities** field for general remarks; the Notes tab is the better place for an ongoing, dated log.

## Tasks

The **Taken** (Tasks) tab lets you create to-do items linked directly to the client, so follow-up actions stay attached to the right record.

When creating a task you can set:

| Field | Notes |
|---|---|
| Name | Required. A short description of the task. |
| Description | Optional longer detail. |
| Deadline | Optional due date. |
| Assigned to | One or more team members in your organisation. |

Tasks move through five statuses:

| Status | Meaning |
|---|---|
| Te doen | To do |
| In behandeling | In progress |
| Wacht op klant | Waiting on the client |
| Afgerond | Completed |
| Geannuleerd | Cancelled |

Use the **Show completed** toggle to include finished tasks in the list. Open tasks are sorted by status and deadline so the most urgent work surfaces first.

## The dossier

The **Dossier** tab is a document store attached to the client. Use it for contracts, signed agreements, identity documents, or any file you want to keep with the record.

<Steps>
  <Step title="Open the upload form">
    On the Dossier tab, click **Add file** to reveal the upload form.
  </Step>
  <Step title="Select files and add details">
    Choose one or more files. Optionally add a **description**, **notes**, and a **category** to keep the dossier organised.
  </Step>
  <Step title="Upload">
    Confirm the upload. Files are stored securely and listed in the dossier immediately.
  </Step>
</Steps>

Supported file types are PDF, Word, Excel, images (JPG, PNG, GIF, WebP), and plain text or CSV. The maximum size is **10 MB per file**.

You can search the dossier by filename, filter by category, edit a file's description or category, download a file, or remove it. Every dossier action is recorded in the client's activity log.

## Per-client settings

Most clients inherit their settings from the administration, but you can override a few per client on the edit form:

- **Voorkeurstaal documenten** (Document language) — the language used for invoices and emails sent to this client. Falls back to the administration's default language, then the platform default.
- **Default VAT percentage** — overrides the administration's default rate for this client's lines.
- **BTW verlegd / BTW vrijgesteld** — reversed-charge or exempt VAT, usually set automatically from the country.
- **G-rekening settings** — if your administration uses a G-account, you can override the percentage and IBAN for this client.
- **Excluded invoicing days** — days of the week on which this client should not be automatically invoiced.

<Note>
  Leave an override blank to keep using the administration default. The client form shows the inherited value as a placeholder so you always know what applies.
</Note>

## Deactivating, restoring, and reactivating a client

How a client is removed depends on whether it has history:

- **No invoices or subscriptions** — the client is permanently deleted.
- **Has invoices or subscriptions** — the client is *deactivated* instead of deleted, so existing invoices stay valid and accessible. A deactivated client moves to the background of the clients list.

To bring a client back:

- **Reactivate** a deactivated client from its detail page — it returns to the active list.
- **Restore** a deleted client if it was soft-deleted, recovering the full record.

<Warning>
  Deactivating a client never deletes invoices, subscriptions, or dossier files. All historical data remains intact.
</Warning>

## Clients across administrations

If your account has more than one administration, each client belongs to one administration. Use the **All clients** overview to see clients across every administration at once. When creating a client from that view, you choose which administration it belongs to.
