If you are switching to TRONVoice from another invoicing tool, you do not have to retype your data. The import wizard reads an Excel or CSV file, recognises common export formats, and lets you review every row before it is created.

## Starting an import

Go to **Importeren** (Import) in the main navigation, or use the **Import** button on the clients or products list. The wizard walks through five steps:

<Steps>
  <Step title="Choose what to import">
    Select **Klanten** (Clients) or **Producten** (Products). Each type has its own set of fields.
  </Step>
  <Step title="Upload your file">
    Upload an `.xlsx`, `.xls`, or `.csv` file, up to 10 MB. For a client import, also select the **administration** the clients belong to. If your account has only one administration, it is selected automatically.
  </Step>
  <Step title="Map columns (if needed)">
    TRONVoice tries to detect the format automatically. If it recognises the file, it skips straight to the preview. If not, you map each column in your file to a TRONVoice field.
  </Step>
  <Step title="Review the preview">
    Check the rows that will be imported. Duplicates are flagged and deselected by default. Select or deselect rows as needed.
  </Step>
  <Step title="Run the import">
    Confirm. TRONVoice creates the selected records and shows a results summary.
  </Step>
</Steps>

## Supported file formats

The wizard accepts **Excel** (`.xlsx`, `.xls`) and **CSV** (`.csv`) files. The first row must contain column headers.

It automatically recognises exports from several popular tools:

When the format is recognised, the column names show in the wizard (for example *WeFact* or *Factuursturen*) and mapping is handled for you. Any other file is treated as a generic spreadsheet and goes through manual column mapping.

## Column mapping

If the format is not auto-detected — or if you click **Customise mapping** — you map each column in your file to a TRONVoice field. The wizard suggests matches based on the header name and recognises both Dutch and English headers (for example `bedrijfsnaam`, `company`, `naam`, or `company_name` all map to the company name).

For each column you can also see sample values from your file, which makes it easy to confirm you have picked the right field.

<Warning>
  You must map the required field — **Bedrijfsnaam** for clients or **Productnaam** for products — or the wizard will not let you continue. Columns left as *Niet importeren* (Do not import) are skipped.
</Warning>

## Validation and duplicate detection

Before anything is created, the wizard checks your file:

- **File safety** — uploads are scanned and sanitised. Cells that could contain spreadsheet formulas are neutralised, and the file type is verified, not just its extension.
- **Empty files** — a file with no data rows is rejected with a clear message.
- **Duplicate detection** — each row is checked against your existing records.

Duplicates are matched on:

| Import type | Matched on |
|---|---|
| Clients | KvK number, then company name |
| Products | Product name |

Rows flagged as duplicates show the reason (for example *KvK-nummer bestaat al* — KvK number already exists) and are **deselected by default**, so re-running an import will not create copies. Use **Select only new** to import just the rows that are not duplicates, or override individual rows manually.

After the import, a summary shows how many records were created, skipped, or failed.

<Tip>
  Start with a small test file of a few rows to confirm your column mapping is correct, then import the full file.
</Tip>

## Importing from accounting integrations

A spreadsheet import is not the only option. If you use an accounting package, you can import clients directly through an integration instead of exporting a file:

- **Exact Online** — sync clients directly from your Exact Online administration. See [Exact Online](/integrations/exact-online).

An integration import keeps the link to the source system, so records can stay in sync over time — something a one-off spreadsheet import cannot do.
